## Overview

This repository contains the complete implementation code for the paper:

> **Development of a Novel Machine Learning Framework for Sustainable Agricultural Development **
> Abdu Salam *et al.* — *PeerJ Computer Science*, 2024.

The codebase implements a deep-learning super-resolution pipeline that enhances the spatial resolution of Landsat multispectral imagery from 30 m to 7.5 m (×4), and applies the enhanced imagery to agricultural monitoring and groundwater resource assessment across the Al-Qassim region of Saudi Arabia (73,000 km²) over the period 1982–2020.

Three super-resolution methods are compared:

| Method | RMSE ↓ | PSNR (dB) ↑ | SSIM ↑ |
|--------|--------|-------------|--------|
| Bicubic Interpolation (baseline) | 15.32 | 30.56 | 0.815 |
| SRCNN | 10.45 | 34.22 | 0.892 |
| **SRGAN (proposed)** | **8.27** | **36.48** | **0.935** |

## Data Sources

All datasets are publicly available from their respective repositories.
**Do not commit raw data to this repository.** Download and place in `data/raw/` as described below.

| Dataset | Variable | Period | Resolution | Source | DOI / URL |
|---------|----------|--------|------------|--------|-----------|
| Landsat Collection 2 L2 | Surface Reflectance (6 bands) | 1982–2020 | 30 m | USGS | https://earthexplorer.usgs.gov/ |
| GRACE JPL RL06.1 Mascon | Terrestrial Water Storage | 2003–2020 | ~300 km | JPL / eoPortal | https://www.eoportal.org/satellite-missions/grace |
| PlanetScope | Multispectral (4 bands) | Oct 2020, Feb 2020 | 3 m | ESA EO Gateway | https://earth.esa.int/eogateway/missions/planetscope |
| CRU-TS v4.05 | Temperature, Precipitation | 1982–2020 | 0.5° | CRU / Harris et al. | https://doi.org/10.1038/s41597-020-0453-3 |
| PME Met Station | Monthly Rainfall | 1982–2020 | Point (Buraidah) | KAPSARC | https://datasource.kapsarc.org/explore/dataset/average-atmospheric-pressure-in-hpmb-observed-by-pme-met-stations-2009xls0/information/ |

### Downloading the data

**Landsat**
1. Go to https://earthexplorer.usgs.gov/
2. Set the Area of Interest to Al-Qassim, Saudi Arabia (bbox: 43.0°E–45.5°E, 25.5°N–27.5°N).
3. Select: *Landsat* → *Landsat Collection 2 Level-2* → filter cloud cover < 10 %.
4. Download scene directories and place under `data/raw/landsat/`.

**GRACE**
1. Go to https://podaac.jpl.nasa.gov/dataset/TELLUS_GRAC-GRFO_MASCON_CRI_GRID_RL06.1_V3
2. Download the monthly NetCDF files.
3. Place `.nc` files under `data/raw/grace/`.

**CRU-TS**
1. Go to https://crudata.uea.ac.uk/cru/data/hrg/
2. Download `cru_ts4.05.1901.2020.tmp.dat.nc.gz` and `cru_ts4.05.1901.2020.pre.dat.nc.gz`.
3. Extract and place `.nc` files under `data/raw/cru_ts/`.

**PME Met Station**
1. Go to the KAPSARC portal (link above).
2. Download the CSV export.
3. Place the CSV file under `data/raw/pme_station/`.

**PlanetScope**
1. Apply for access via https://earth.esa.int/eogateway/missions/planetscope
2. Download orthorectified 4-band GeoTIFFs for October 2020 and February 2020.
3. Place under `data/raw/planetscope/`.

## Installation

### Prerequisites

- Python 3.10+
- GDAL 3.4 (must be installed at the **system level** before `pip install`)
- CUDA 11.6 + cuDNN 8.4 (for GPU training; see [TensorFlow GPU guide](https://www.tensorflow.org/install/pip))

### Step-by-step

```bash
# 1. Clone the repository
git clone https://github.com/<repository-to-be-confirmed>.git
cd sr_agri

# 2. Create a virtual environment
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate

# 3. Install system-level GDAL (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install -y gdal-bin libgdal-dev python3-gdal

# 4. Install Python dependencies
pip install --upgrade pip
pip install -r requirements.txt

# 5. (GPU users) Install CUDA-enabled TensorFlow
pip install tensorflow[and-cuda]==2.10.1
```

Verify the installation:
```bash
python -c "import tensorflow as tf; print(tf.__version__, tf.config.list_physical_devices('GPU'))"
```

## Usage

### Run the complete pipeline

```bash
python run_pipeline.py
```

This runs all 8 stages sequentially:

| Stage | Description | Typical duration |
|-------|-------------|-----------------|
| 0 | Environment & dependency check | < 1 min |
| 1 | Preprocessing (all 5 datasets) | 30–60 min |
| 2 | SR model training (Bicubic + SRCNN + SRGAN) | 16–20 h (GPU) |
| 3 | Vegetation & water index computation | 15–30 min |
| 4 | Spatiotemporal analysis | 30–60 min |
| 5 | Trend & correlation analysis | 10–20 min |
| 6 | Validation & model comparison | 10–20 min |
| 7 | Figure generation | 5–10 min |

### Run selected stages only

```bash
# Preprocessing only
python run_pipeline.py --stages 1

# Training + evaluation only
python run_pipeline.py --stages 2

# Skip training (use pre-trained checkpoints)
python run_pipeline.py --stages 2 --skip_training

# Analysis + figures (after training is complete)
python run_pipeline.py --stages 3 4 5 6 7 --skip_training
```

### Run individual scripts

Each module can also be run independently:

```bash
# Preprocess Landsat scenes
python preprocessing/landsat_preprocess.py --input data/raw/landsat --output data/processed/landsat

# Train SRGAN
python models/srgan.py --data data/processed/landsat --epochs 200 --batch 16

# Evaluate SRGAN on test set
python models/srgan.py --data data/processed/landsat --output results/srgan --eval_only

# Compute vegetation indices for all SR scenes
python indices/vegetation_indices.py --sr_images results/srgan --output results/indices

# Generate all figures
python visualisation/plot_results.py --analysis results/analysis --output figures
```

## Model Architecture Summary

### Bicubic Interpolation (Equation 1)
Standard bicubic kernel with `a = −0.5`. Used as the performance baseline.

### SRCNN — Super-Resolution CNN (Equation 2)
Three-layer convolutional network applied to bicubic-upsampled input:
- Layer 1 — Patch extraction: Conv(9×9, 64) + ReLU
- Layer 2 — Non-linear mapping: Conv(1×1, 32) + ReLU
- Layer 3 — Reconstruction: Conv(5×5, *C*) [linear]

### SRGAN — Super-Resolution GAN (Equation 3)
**Generator**: Modified ResNet with 16 residual blocks + 2× sub-pixel convolution (×4 total upscaling)

**Discriminator**: VGG-style binary classifier (8 convolutional blocks)

**Loss**: Perceptual (VGG-19 `block5_conv4`) + adversarial + pixel MSE

**Training**: 10-epoch pixel-loss warm-up → 200-epoch adversarial training

## Vegetation Indices

| Index | Equation | Reference |
|-------|----------|-----------|
| NDVI | (NIR − RED) / (NIR + RED) | Rouse et al., 1974 |
| SAVI | (1 + 0.5) × (NIR − RED) / (NIR + RED + 0.5) | Huete, 1988 |
| MSAVI2 | (2·NIR + 1 − √[(2·NIR+1)² − 8(NIR−RED)]) / 2 | Qi et al., 1994 |
| NDWI | (GREEN − NIR) / (GREEN + NIR) | McFeeters, 1996 |
| MNDWI | (GREEN − SWIR1) / (GREEN + SWIR1) | Xu, 2006 |
